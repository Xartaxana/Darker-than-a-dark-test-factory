# /qa-loop — один проход конвейера

Запусти агента **qa-orchestrator** (через Task, subagent_type: qa-orchestrator).
Он выполнит один проход по `state/rules.yaml`: просканирует `runs/`, `bugs/`,
`test-cases/`, `state/app-under-test.yaml`, применит правила по приоритету, поставит
локи и передаст работу профильным агентам, затем снимет локи и запишет решения в
`state/orchestrator-log.md`.

Шаги:
1. Если `$ARGUMENTS` содержит `--dry-run` — попроси оркестратора только показать, какие
   правила сработали бы, без диспетчеризации.
2. Иначе запусти полный проход.
3. По завершении покажи пользователю сводку из `state/orchestrator-log.md` (что
   запускалось и с каким исходом) и текущее состояние очереди (открытые баги,
   прогоны в NeedsTriage, кейсы к автоматизации).

Границы: ни один агент не правит код в `app-under-test/`. Статусы `Blocked` и лок
`wip` пропускаются — это точки для человека.
