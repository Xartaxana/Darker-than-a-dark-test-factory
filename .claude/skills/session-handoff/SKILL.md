# session-handoff — проверка переноса сессии (D-0050, порт из OS-репо)

Принцип: handoff проверяет вечером ровно то, что следующая сессия
прочитает утром. Утренний путь AO3: `CLAUDE.md` (автозагрузка) →
`docs/HANDOFF.md` («Читать первым при старте», resume-заметки +
таблица «что где») → `state/factory-status.md` (ГЕНЕРИРУЕТСЯ
`scripts/queue_snapshot.py`, ручные числа запрещены — правило A4,
docs/09-improvement-plan.md). Journal: `logs/routing-log.jsonl`,
пишется ТОЛЬКО через `scripts/log_append.py`.

## Шаги

1. **Git.** `git status --short` пуст — незакоммиченное коммитится
   сейчас (узким явным списком путей, не `git add -A`); незнакомые
   untracked-файлы не заметать в коммит молча — разобраться, чьи они
   (возможна параллельная сессия, D-0060 OS-репо / F-23 AO3). Затем
   `git log origin/master..master --oneline` пуст — иначе `git push`.
   Если сессия трогала OS-репо (`D:\Improving_AI\Operating-System-for-LLMs`)
   — та же проверка там (ветка `main`).
2. **Журнал закрыт** (`logs/routing-log.jsonl`): прогнать
   `python scripts/log_append.py open-dispatches` — каждый показанный
   открытый `delegated` сверить с реальностью (D-0076 OS-репо / F-44
   ЗДЕСЬ): воркер по `worker_ref` жив или результат ждёт приёмки —
   иначе фантом (запись без запуска); фантом закрывается пометкой в
   notes следующего события и, если работа нужна, честным
   ре-диспатчем. `lead_degraded` без `lead_restored`
   допустим ТОЛЬКО как ПОСЛЕДНЕЕ событие журнала (деградация,
   переживающая сессию, D-0039 OS-репо) — иначе следующая сессия
   стартует ошибочно на полном Lead. События сессии написаны
   `scripts/log_append.py`, не ручной правкой файла (permission
   hygiene, CLAUDE.md п.4).
3. **Живое состояние.**
   - `docs/HANDOFF.md`: блок «Где мы» и resume-заметки отражают
     фактическое закрытие сессии (не хвост предыдущей).
   - `state/factory-status.md`: перегенерирован
     `scripts/queue_snapshot.py` ПОСЛЕ последнего изменения
     артефактов сессии — сверить `generated_at` со временем последних
     правок `bugs/`/`test-cases/`/`runs/`; ручные числа в
     HANDOFF/докках запрещены (A4). Расхождение счётчиков —
     FAIL, чинится регенерацией скриптом, не руками.
   - `state/coverage-map.md`: перегенерирован `scripts/coverage_map.py`
     ПОСЛЕ последнего изменения `test-cases/**` — если нет, перегенерировать
     сейчас; шапка файла не несёт строку «⚠ реестр фич протух» без причины
     (детектор сравнивает `docs/feature-registry.yaml:inventoried_at_commit`
     с `state/app-under-test.yaml:source_commit` — расхождение живёт до
     переинвентаризации, не является само по себе FAIL хендоффа, но
     отмечается в резюме сессии, если появилось за эту сессию).
   - Эмуляторная очередь пуста / эмулятор и Appium погашены —
     правило записано в `docs/environment-setup.md:211-214` (секция
     «Стабильность хоста: эмулятор + локальный GPU-инференс не
     совмещать», строка 213: «дополнительная мотивация гасить
     эмулятор при пустой эмуляторной очереди»); сверка присутствия —
     каноническая форма `. scripts/tasks.ps1; Get-Device` (не голый
     `adb`, командная гигиена CLAUDE.md п.6).
   - Кросс-деплойные очередь-пункты (D-0082 OS-репо, порт 2026-07-19,
     CLAUDE.md 4б): всё, что сессия адресовала OS-репо, записано в их
     `CURRENT_CONTEXT.md` кросс-коммитом (и наоборот: пункты OS→AO3 —
     в этом HANDOFF); пункт только в notes журнала/FINDINGS = не
     существует (класс F-48, 4 испарившихся прецедента калибровки №3).
   - Тест переноса: «всё, что знаю только я, записано в файлы?»
4. **Boot-бюджет.** Суммарный размер в байтах `CLAUDE.md` +
   `docs/HANDOFF.md` + `docs/09-improvement-plan.md` (в бут-перечне с
   2026-07-17; его нарративы обязаны уходить в docs/09-history.md —
   рост плана = кандидат свипа). Сравнить с прошлым замером (в сообщении
   прошлого handoff-коммита; при смене состава перечня прошлый замер
   несравним — новый baseline, явно пометить в коммите). Флаги:
   рост >10% за сессию без объяснимой причины; суммарно >100 КБ →
   пункт «boot-диета» в `docs/09-improvement-plan.md`, если его там
   ещё нет. (Ось платного автозагружаемого контекста — чек 10
   еженедельной калибровки OS-репо, WEEKLY_CALIBRATION_PROTOCOL.md.)
   Порядок реакции на пробой (порт OS-репо 2026-07-10, D-0068):
   СНАЧАЛА архивный свип — закрытое/нарративное уводится в архивный
   дом (HANDOFF-история, docs/) со сверкой, что урезанная версия не
   разошлась с полной и указатели живы; глубокие срезы оперативных
   домов (CLAUDE.md, действующие правила) — НЕ в ходе handoff, а
   отдельным решением координатора+человека пунктом очереди.
5. **Живость утренней цепочки.** Существуют: `CLAUDE.md`,
   `docs/HANDOFF.md`, `state/factory-status.md`, `state/rules.yaml`
   (переименование любого без правки HANDOFF.md = сломанное утро
   следующей сессии).
6. **Итог.** Короткий отчёт по пунктам 1–5 (OK/FAIL и что сделано).
   FAIL чинится ДО закрытия сессии, не записывается «на потом».
   Финальный коммит (в сообщении — замер п.4 как baseline для
   следующего handoff) и push — последнее действие сессии.

## Детектор пропуска

Утреннее правило Session Start в `docs/HANDOFF.md` (первые строки
после «Читать первым при старте»): `git status --short` и
`git log origin/master..master --oneline` на старте новой сессии.
Грязное дерево или расхождение с origin в этот момент = прошлая
сессия закрылась без handoff-проверки — фиксируется находкой через
`log_append.py` (по словарю событий CLAUDE.md; минимум — пометка в
notes ближайшего события журнала; `defect_found` — только если
находка ссылается на конкретную ПРИНЯТУЮ задачу, его `ref`
обязателен), не поглощается молча. Внешняя сеть — еженедельная
калибровка OS-репо читает журналы ОБОИХ репо (routing-log.jsonl
здесь и в OS-репо), так что несловленный пропуск здесь виден и там.
