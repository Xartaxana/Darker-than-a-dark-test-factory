# exploratory-charters/ — исследовательские сессии как артефакты (P7 ревью docs/10)

Charter — план и протокол одной exploratory-сессии. Scripted-регрессия ловит
известное; charter ищет неизвестное: новые риски, баги вне кейсов, пробелы
дизайна. Charter — первоклассный артефакт фабрики, как test-case и bug.

## Конвенции

- Один файл `CH-NNN.md` = одна сессия с таймбоксом. ID `CH-NNN` уникален и
  не переиспользуется (max+1).
- Шаблон — `docs/templates/charter.md`; исполняет агент `exploratory-tester`;
  заводит агент `charter-designer` (или Lead/человек напрямую).
- Статусы: `Proposed` (автозаведён charter-designer, ждёт гейта
  критик-на-план) → `Planned` (план принят: `plan_review` заполнен
  вердиктом PASS; прямое заведение Lead/человеком стартует отсюда) →
  `InProgress` (лок взят, сессия идёт) → `Done` (протокол и follow-up
  заполнены, `mission_leftover` заполнен). `Blocked` — критик FAIL /
  деградация. Схема — `schemas/charter.schema.yaml`, машина статусов —
  `schemas/transitions.yaml` (шаг 2 внедрён 2026-07-21).
- Генерация следующих сценариев — комбинаторная, по `PERTURBATIONS.md`
  (матрица «зона × возмущение» + seam-пары; владелец записи —
  charter-designer): новый чартер берёт непокрытую клетку или остаток
  миссии (`mission_leftover`) новейшего Done-чартера.
- Находки НЕ живут только прозой протокола (класс AT-BUG-004/005/006 —
  «знание в теле артефакта, которого не видит конвейер»): каждый
  воспроизводимый баг → через bug-reporter в `bugs/` (ссылка в frontmatter
  `found_bugs`), каждый пробел покрытия → follow-up TC через test-designer
  (ссылка в `followup_tc`); новый риск → предложение в docs/01 §10.
- Вложения (скриншоты, дампы) — `exploratory-charters/attachments/CH-NNN/`,
  не корень репо.

## Триггеры заведения charter'а (docs/10 P7, docs/09 Этап 4 п.11)

С 2026-07-21 заведение АВТОМАТИЧЕСКОЕ — правило rules.yaml «Завести
следующий exploratory-чартер» диспатчит charter-designer, когда очередь
чартеров пуста и:

- каденция: новейший Done старше 72 ч (cadence 3 дня, слово оператора);
- вердикт APP_CHANGED в триаже после новейшего Done;
- кластер багов: >=2 новых Open|Reopened app_bug одной зоны
  (эвристика «где один — там два»).

Гейт: Proposed → Planned только через критик-на-план (правило «Ревью
плана Proposed-чартера»; вердикт в `plan_review`). Прямое заведение
Lead/человеком (новая крупная функция, перед релизом — сразу Planned)
остаётся легальным. Метрики (`charters_executed`, `bugs_per_charter`,
`new_tc_from_charters`) считает queue_snapshot из frontmatter этого
каталога; деградация генератора (K чартеров подряд без находок
bug/test-gap = повтор покрытого) — чек еженедельной калибровки.
Детектор отказа заведения — sla_sweep чек `charter_queue_empty` (96 ч).
