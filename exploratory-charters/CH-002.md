# CH-002 — Хрупкая зона темы: остаток миссии CH-001 под Appium-драйвом

## Mission
CH-001 (Done, 2026-07-14) покрыл state/system-туры зоны темы, но таймбокс и
инструмент (слепой adb-драйв без Appium) оставили ровно те ветки, где ждём
гонки: прерывания посреди смены темы, персист-гонка kill-after-toggle,
поведение FORCE_DARK на Cloudflare-интерстишле, бейджи под сменой темы.
Урок CH-001: слепые chained-тапы через меняющийся layout Cloudflare-релончей
ненадёжны (дисквалифицированный трайл H4, п.9 протокола) — эта сессия идёт
С APPIUM-ВЕРИФИКАЦИЕЙ каждого шага (живое дерево между действиями), иначе
«потеря записи» неотличима от промаха тапа.

## Scope
- **In (остаток миссии CH-001, дословно из его Follow-up):**
  1. interruptions-тур: поворот/сворачивание ПОСРЕДИ смены темы и жеста
     яркости (в CH-001 не начат);
  2. H4 персист-гонка: kill сразу после смены темы, каждый шаг верифицирован
     деревом/prefs ДО force-stop (снимает tooling-класс CH-001);
  3. H2 на Cloudflare-интерстишле: смена темы ВО ВРЕМЯ challenge
     (live-режим отдаёт интерстишл на каждом холодном старте — CH-001 п.2);
  4. жест яркости и его прерывания — мультитач W3C Actions через Appium
     (adb `input` не умеет синхронный two-finger drag);
  5. H1-стресс: матрица 6 переходов Light/Dark/System × 2 входа (side panel
     / Settings) с БЫСТРЫМ чередованием (CH-001 сделал 1 спокойный проход);
  6. seed-бейджи: пережил ли рендер рейтинг-бейджей смену темы (нужен
     seed + replay — в CH-001 не поднимался без Appium).
- **Out:** штатные пути смены темы (TC-047/048/049/050/054 покрывают);
  сами рейтинги/фильтры/скачивания — только как фон; повтор state/system-
  туров CH-001 (уже покрыты, консистентность подтверждена).

## Риски и гипотезы
- Прерывание (поворот/сворачивание) посреди применения темы оставляет
  смешанный рендер (нативный тёмный / WebView светлый) или теряет запись
  prefs (R-11: «Dark mode has broken four times»).
- Kill в окне между toggle и записью `theme_mode` теряет выбор (H4 CH-001 —
  не воспроизведено надёжно, НЕ опровергнуто).
- Смена темы во время Cloudflare-challenge ломает ALGORITHMIC_DARKENING на
  интерстишле или на странице после него.
- Быстрое чередование двух входов (панель↔Settings) рассинхронизирует общий
  стейт (H1 CH-001 — один спокойный проход, стресса не было).
- Бейджи рейтингов (background-color `[data-ao3-rate-btn]`) после смены
  темы становятся нечитаемыми/пропадают.

## Эвристики / туры
- interruptions-тур: rotate/home/recents/kill в момент анимации смены темы
  и посреди двухпальцевого жеста яркости.
- race-тур (H4): toggle → НЕМЕДЛЕННЫЙ force-stop, с Appium-фиксацией
  дерева и чтением prefs до/после; варьировать задержку 0/100/300 мс.
- timing-тур (H2): холодный старт live → на интерстишле сменить тему обоими
  входами; после прохождения challenge проверить рендер.
- stress-тур (H1): 6 переходов × 2 входа быстрым чередованием, проверка
  консистентности обоих входов и prefs после каждого раунда.
- визуальный тур: `webview_avg_luma`-прокси (реализован для TC-048/055) +
  скриншоты бейджей до/после каждой смены.

## Data setup
- Поднять окружение канонически: Start-Emulator → Start-Appium → Install-App
  (tasks.ps1); драйв — ЧЕРЕЗ Appium-сессию (живое дерево
  `python scripts/ui_snapshot.py`, W3C Actions для мультитача), НЕ слепой adb.
- Для тура 6 (бейджи): replay-режим + seed работ с рейтингами (фикстуры
  фреймворка; runbook CA в HANDOFF — `Start-Emulator -WritableSystem`).
  Туры 1-5 легальны в live (интерстишл нужен именно live, тур 3).
- Чистые prefs на старте интеррапшн/race-туров (`pm clear`) — фиксируем
  дефолт, как CH-001 п.2.
- Ловушка из CH-001 (осевое наблюдение): свёрнутый handle side panel
  отзывчив только по видимой иконке (x≤~37), НЕ по всей заявленной
  a11y-области — тапать по content-desc-ноде через Appium, не по центру
  области.
