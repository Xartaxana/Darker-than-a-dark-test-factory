# TC-019 — Отмена Clear all ratings сохраняет данные

## Предусловия
- Приложение запущено, в Room засеяны работы с рейтингами
  (`framework/data/works.py::ALL`).
- Открыт экран Settings, диалог подтверждения «Clear all ratings» открыт (кнопка
  «Clear…» нажата).

## Сценарий (Given-When-Then)

**Given** приложение запущено, в библиотеке есть засеянные работы с рейтингами
**And** на экране Settings открыт диалог подтверждения «Clear all ratings»

**When** пользователь нажимает «Cancel»/закрывает диалог, не подтверждая

**Then** диалог закрывается без удаления
**And** экран Library по всем рейтинговым вкладкам по-прежнему содержит засеянные
работы (данные в БД не изменены)

## Проверяемые данные
| Параметр | Значение |
|---|---|
| Работы | `framework/data/works.py::ALL` (5 работ, по одной на каждый рейтинг) |

## Заметки для автоматизации
- Переиспользует локатор открытия диалога из TC-018/TC-004
  (`settings_screen.open_clear_all_dialog`); дополнительно требуется локатор кнопки
  отмены — вывести из живого дерева диалога (код → дерево → скриншот, см.
  framework/README.md).
