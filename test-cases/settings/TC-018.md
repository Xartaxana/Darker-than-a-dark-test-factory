# TC-018 — Диалог подтверждения перед Clear all ratings

## Предусловия
- Приложение запущено, в Room засеяны работы с рейтингами (`framework/data/works.py`
  `ALL`, через `seed_db.py`).
- Открыт экран Settings.

## Сценарий (Given-When-Then)

**Given** приложение запущено, в библиотеке есть засеянные работы с рейтингами
**And** открыт экран Settings

**When** пользователь нажимает кнопку «Clear…» (секция Data)

**Then** появляется диалог подтверждения (AlertDialog) с текстом про очистку всех
рейтингов
**And** данные в БД остаются нетронуты, пока диалог не подтверждён (засеянные работы
всё ещё в Library)

## Проверяемые данные
| Параметр | Значение |
|---|---|
| Работы | `framework/data/works.py::ALL` (5 работ, по одной на каждый рейтинг) |

## Заметки для автоматизации
- Кнопка «Clear…» с юникод-многоточием; клик в Compose висит на родителе текстового
  узла — см. `settings_screen.open_clear_all_dialog` (уже используется в TC-004).
- Этот кейс проверяет только факт появления диалога и отсутствие побочного эффекта
  до подтверждения — не дублирует TC-004 (Automated, area=smoke: полный цикл
  подтверждение→очистка) и не дублирует TC-019 (отмена).
