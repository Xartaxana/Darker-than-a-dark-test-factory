# TC-047 — Переключение Dark применяется мгновенно, Activity не пересоздаётся

## Предусловия
- Приложение запущено с чистыми данными, тема = Light (или System при системной
  светлой теме), на экране Browse открыта вкладка с загруженной страницей AO3.
- Открыт экран Settings.

## Сценарий (Given-When-Then)

**Given** приложение в светлой теме, на вкладке Browse отображается загруженная
страница

**When** пользователь в Settings выбирает тему "Dark"

**Then** нативные Compose-экраны немедленно перекрашиваются в тёмную схему (без
видимого пересоздания/мигания Activity — состояние экрана, включая открытые вкладки
и их скролл, сохраняется)
**And** после возврата на экран Browse открытые вкладки/URL/скролл остаются теми же,
что были до переключения (подтверждает отсутствие recreation)

**Инвариант:** переключение темы (Light→Dark) не меняет состояние навигации
экрана Browse — URL открытой вкладки И позиция прокрутки страницы после
переключения совпадают со значениями до переключения (Activity не
пересоздаётся, ни один аспект состояния не теряется).

## Проверяемые данные
| Параметр | Значение |
|---|---|
| Исходная тема | Light |
| Целевая тема | Dark |
