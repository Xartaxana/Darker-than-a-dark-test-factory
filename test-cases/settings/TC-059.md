# TC-059 — System-тема: WebView следует за системной сменой uiMode без in-app toggle

## Предусловия
- Приложение запущено с чистыми данными (`pm clear`), в Settings выбрана тема
  "System".
- Системная тема ОС установлена в Light (`adb shell cmd uimode night no`).
- На вкладке Browse открыта загруженная страница AO3 (WebView содержимое видно
  светлым).
- Режим прогона: replay либо live-smoke (тот же режим, что TC-048/TC-049).

## Сценарий (Given-When-Then)

**Given** тема приложения = System, системная тема ОС = Light, на Browse открыта
загруженная страница AO3, содержимое WebView светлое

**When** системная тема ОС переключается на Dark (`adb shell cmd uimode night yes`)
**без** какого-либо действия внутри приложения (без захода в Settings, без тапа
Contrast в side panel)

**Then** содержимое WebView (сама веб-страница AO3) переходит в тёмную цветовую
схему вслед за системной сменой — так же, как нативный UI (side panel/Settings,
см. TC-049) — без in-app toggle и без холодного рестарта приложения
**And** обратное переключение (`cmd uimode night no`) возвращает содержимое WebView
в светлую схему — эффект воспроизводится в обе стороны
**And** активная вкладка Browse и её URL не меняются при обоих переключениях
(смена темы не пересоздаёт WebView и не роняет навигацию)

**Инвариант:** контент WebView под режимом темы System всегда согласован с текущим
системным uiMode (dark<->light), независимо от направления и количества
переключений, без in-app действия — тот же общий стейт `darkTheme`, что применяется
к нативному UI (частный случай инварианта «смена темы не меняет URL, активную
вкладку и сохранённые данные»: наблюдаемое здесь переключение WebView-рендера не
затрагивает URL/вкладку/данные).

## Проверяемые данные
| Параметр | Значение |
|---|---|
| Тема приложения | System (не меняется внутри приложения) |
| Системная тема | Light → Dark → Light через `adb shell cmd uimode night yes/no` |
| Наблюдаемый прокси | `webview_avg_luma` (тот же прокси, что TC-048/TC-055) |

## Заметки для автоматизации
- Отличается от TC-049 (та проверяет ТОЛЬКО нативный Compose UI под System и явно
  исключает WebView, см. её «Заметки») и от TC-048 (та проверяет WebView, но триггер
  — in-app toggle Settings, не системная смена). Этот кейс — пересечение: System +
  WebView + внешний (не in-app) триггер. Не смешивать assert'ы всех трёх кейсов в
  одном тесте.
- Наблюдаемость WebView — тот же прокси `webview_avg_luma`, что в TC-048/TC-055
  (`driver.get_screenshot_as_png()` + Pillow `.convert("L").histogram()`), с тем же
  порогом и той же логикой поллинга до перерисовки после программного `reload()`
  (`LaunchedEffect(darkTheme)` в BrowserScreen.kt срабатывает на любое изменение
  `darkTheme`, включая пришедшее из `isSystemInDarkTheme()` при рекомпозиции по
  config-change `uiMode`, а не только из ручного toggle) — блокера/новой фикстуры
  не требуется, прокси уже реализован и используется в двух существующих
  автотестах.
- Системная тема — общий ресурс эмулятора: тест обязан выставить Light на входе и
  вернуть Light в `finally`, как это уже делает TC-049 (`set_night_mode` в
  core/adb, `set_system_dark_mode` в steps/app_steps) — переиспользовать те же
  шаги, не дублировать обёртку.
- Тайминг: после `cmd uimode night yes/no` рекомпозиция и `wv.reload()`
  асинхронны — поллинг `webview_avg_luma`, не мгновенное чтение (см. TC-048,
  тот же паттерн).
- Ловушка локатора (CH-001, находка «замеченные аналоги»): если сценарий по ходу
  теста взаимодействует со свёрнутым handle side panel, тап координатой по центру
  заявленной clickable-области `[0,496][84,664]` (x≈42) НЕ отзывчив — проваливается
  в WebView; надёжен только тап по content-desc-ноде «Expand panel»
  `[5,564][37,596]` (Appium через `find_element` по content-desc, не coord-based
  тап по центру bounds). Этот кейс сам side panel не раскрывает (наблюдение только
  через Browse), но заметка актуальна для любого соседнего/будущего кейса этой
  области, использующего свёрнутый handle.
- Источник: exploratory-charters/CH-001.md, находка 1 (System-тур, скриншоты
  07-10 в attachments/CH-001/), воспроизведено вручную в обе стороны.
