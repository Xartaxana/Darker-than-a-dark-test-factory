# TC-057 — Side panel Home переводит активную вкладку на главную AO3

## Предусловия
- Приложение запущено с чистыми данными, вкладка Browse активна.
- Активная вкладка открыта НЕ на главной странице AO3 (например, на листинге с
  фильтром/страница работы) — достигается обычной навигацией внутри WebView перед
  началом сценария.
- Side panel развёрнут (тап по handle) и показывает иконку Home с
  contentDescription «AO3 home».
- Режим прогона: live-smoke (Home ведёт на реальный фиксированный URL
  `https://archiveofourown.org`, тот же, что открывает P0-смоук при холодном старте)
  либо replay с записью, покрывающей корневой URL (`ao3_home_smoke.mitm`).

## Сценарий (Given-When-Then)

**Given** активная вкладка Browse открыта на странице, отличной от главной AO3, side
panel развёрнут и показывает иконку Home

**When** пользователь нажимает иконку Home в side panel

**Then** WebView активной вкладки загружает `https://archiveofourown.org` (URL
активной вкладки становится равен главной странице AO3)
**And** side panel сворачивается автоматически (без отдельного действия пользователя)

## Проверяемые данные
| Параметр | Значение |
|---|---|
| Исходная страница активной вкладки | любая страница AO3 кроме главной |
| Ожидаемый URL после нажатия | `https://archiveofourown.org` (`BrowserViewModel.HOME_URL`) |
| Ожидаемое состояние панели после | свёрнута (`panelExpanded=false`) |

## Заметки для автоматизации
- Локатор кнопки — по contentDescription «AO3 home» (иконка `Icons.Outlined.Home`,
  не меняется в зависимости от состояния — в отличие от Fullscreen/Contrast, где текст
  переключается).
- `onNavigateHome` не меняет WebView напрямую: пишет `pendingNavigationUrl` в
  `BrowserViewModel.uiState` (`navigateActiveTabTo`), которое отдельным
  `LaunchedEffect` в `BrowserScreen.kt` (~212-218) применяет через
  `webViews[activeId]?.loadUrl(url)` и сразу очищает (`clearPendingNavigation`) —
  проверка URL должна дождаться завершения загрузки (`wait_until`), а не читать
  `getCurrentUrl()` сразу после тапа (гонка, тот же класс, что был устранён в
  `browser_steps.assert_blurb_hidden`/`AT-BUG-004`).
- Наблюдаемость URL — через `driver.current_url` в WEBVIEW-контексте (уже
  используется существующими web-степами) либо через факт того, что на экране
  отрендерилась главная страница AO3 (заголовок/структура листинга по умолчанию),
  если прямое чтение URL недоступно в данном контексте драйвера.
- Не требует новой replay-записи сверх уже существующей `ao3_home_smoke.mitm` — Home
  ведёт на тот же корневой URL, что эта запись покрывает; фикстур/сидинга сверх
  обычного `clean_app` не требуется. Не блокер.
