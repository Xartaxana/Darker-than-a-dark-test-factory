# TC-010 — Мгновенное обновление бейджа (вход: панель работы)

## Предусловия
- Приложение запущено с чистыми данными.
- Открыта страница работы `/works/{id}`, панель `RatingMenu` раскрыта, рейтинг ещё
  не выставлен.

## Сценарий (Given-When-Then)

**Given** приложение запущено и открыта страница работы `/works/{id}` без рейтинга
**And** зафиксирован факт, что WebView не совершал навигацию/reload с момента
открытия страницы (baseline: URL и `onPageFinished` не срабатывали повторно)

**When** пользователь ставит рейтинг «Loved» через панель `RatingMenu`

**Then** бейдж на странице (в панели и/или инжектированный визуальный индикатор)
отражает «Loved» немедленно
**And** WebView не выполнил повторную навигацию/reload (тот же `onPageFinished`
event count / та же history entry, без нового page load)

## Проверяемые данные
| Параметр | Значение |
|---|---|
| Работа | `ao3_id` из `framework/data/works.py` |

## Заметки для автоматизации
- Технически проверка "без reload" — через отсутствие нового `onPageFinished` /
  неизменный WebView history index, либо через отсутствие визуального flash/loading
  indicator; заранее согласовать с test-automator конкретный сигнал в
  `core/waits`/`web/` слое.
- Источник поведения: `applyRating`/`savePanelRating` в BrowserViewModel вызывают
  `window.applyRatings(...)` через `evaluateJavascript`, что не является навигацией.
- Не дублирует TC-007 (там основной фокус — успешная простановка каждого из 5
  рейтингов и попадание в Library); здесь фокус узкий — именно факт "без
  перезагрузки", включая техническую метрику отсутствия reload.
