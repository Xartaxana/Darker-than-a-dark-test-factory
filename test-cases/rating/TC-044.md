# TC-044 — Note-кнопка открывает overlay с раскрытым комментарием

## Предусловия
- Приложение запущено, в Room засеяна работа W с рейтингом Liked и непустым
  `comment` (существующая заметка) — либо создана заранее через UI при подготовке
  фикстуры.
- Открыта листинговая страница, содержащая работу W; карточка показывает Note-кнопку
  (карандаш), т.к. у работы уже есть комментарий.

## Сценарий (Given-When-Then)

**Given** работа W на листинге имеет существующий комментарий и видимую Note-кнопку
(карандаш, слева от Rate-кнопки)

**When** пользователь нажимает Note-кнопку работы W

**Then** открывается overlay рейтинга для работы W (`RatingOverlay`) с полем
комментария, уже развёрнутым (не свёрнутым/скрытым за доп. действием)
**And** поле комментария предзаполнено существующим текстом заметки

## Проверяемые данные
| Параметр | Значение |
|---|---|
| Работа W | rating=LIKE, comment="Existing note text" |

## Заметки для автоматизации
- Note-кнопка отображается только когда у работы есть непустой `comment` (см.
  ao3_bridge.js: JS injects/removes Note buttons per work) — фикстура обязана
  включать непустой comment, иначе кнопка не появится и локатор не найдётся.
- Требует того же расширения сидинга под `comment`, что в TC-043, либо подготовку
  через реальный UI flow заранее (тап Rate → Liked → ввод заметки → сохранить, затем
  начать собственно тестируемый сценарий сначала).
