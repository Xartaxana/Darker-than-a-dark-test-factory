# TC-045 — Личные теги не скрывают работу

## Предусловия
- Приложение запущено, в Room засеяна работа W с рейтингом Disliked и непустым
  списком личных тегов (`tags=["spoiler", "reread-candidate"]`) — требует расширения
  сидинг-скрипта под непустой `tags` (JSON-конвертер Room), см. заметки.
- В Settings включена фильтрация с Disliked в hidden-set (дефолт).
- Открыта листинговая страница, содержащая работу W.

## Сценарий (Given-When-Then)

**Given** работа W имеет rating=Disliked (скрывается фильтрацией по правилу
видимости) и непустые личные теги

**When** пользователь просматривает листинговую страницу с включённой фильтрацией

**Then** работа W скрыта на листинге — как и любая другая Disliked-работа (личные
теги никак не меняют этот факт, ни ослабляя, ни усиливая скрытие)
**And** переход на экран Library, вкладка DISLIKED, показывает работу W с
сохранёнными личными тегами (видны через индикатор/детали карточки)

## Проверяемые данные
| Параметр | Значение |
|---|---|
| Работа W | rating=DISLIKE, tags=["spoiler", "reread-candidate"] |
| Фильтрация | Enable filtering = ON, Disliked в hidden-set |

## Заметки для автоматизации
- Цель кейса — доказать отсутствие эффекта: результат видимости идентичен тому, что
  было бы без тегов вообще (ссылка на существующее покрытие Disliked-скрытия в
  TC-013/TC-014/TC-015, area=visibility, P0). Не дублирует их — там нет тегов в
  фикстуре, здесь специально добавлены, чтобы исключить регресс "теги случайно стали
  влиять на видимость".
- `seed_db.py::_insert_rows` пишет `tags=None` жёстко — расширение сидинга должно
  поддержать JSON-сериализованный список (см. `Converters.kt` в app-under-test для
  формата хранения — не изменять сам код приложения, только формат INSERT в тестовом
  сидинге должен ему соответствовать).
- Подсветка совпадающих AO3-тегов индикатором (вторая часть требования про личные
  теги) — не проверяется в этом кейсе (отдельный аспект UI, не про видимость); при
  необходимости можно завести отдельный P2/P3-кейс на индикатор подсветки, сейчас не
  запрошено явно в §9 для видимости.
