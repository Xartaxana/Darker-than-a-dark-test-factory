# TC-029 — Фильтр по фандому

## Предусловия
- Приложение запущено, в Room засеяны 5 работ с одним рейтингом и разными фандомами
  (`framework/data/works.py::ALL` — Fandom Alpha/Beta/Gamma/Delta/Eps).
- Открыт экран Library, соответствующая вкладка, фильтр не применён.

## Сценарий (Given-When-Then)

**Given** на вкладке Library видно 5 работ с разными значениями фандома

**When** пользователь в фильтр-панели выбирает/вводит фандом "Fandom Alpha"

**Then** список показывает только работу с fandom="Fandom Alpha" (1 работа)
**And** остальные 4 работы с другими фандомами скрыты

## Проверяемые данные
| Параметр | Значение |
|---|---|
| Работы | `ALL` из `framework/data/works.py`, один рейтинг |
| Фильтр фандома | "Fandom Alpha" (точное совпадение с LOVED) |

## Заметки для автоматизации
- Как и в TC-027/TC-028, для одной вкладки все 5 работ нужно засеять с одинаковым
  рейтингом — параметризовать `seed_db.seed()`.
- Проверить точность совпадения (точное строковое или подстрока) по факту UI —
  PROJECT.md не уточняет; если поле реализовано как текстовый ввод с частичным
  совпадением, скорректировать Then при обнаружении расхождения (не критично для
  сути кейса — сам факт сужения списка).
