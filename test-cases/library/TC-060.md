# TC-060 — Личные теги: фильтр по AND, не по OR

## Предусловия
- Приложение запущено, в Room засеяны 3 работы с ОДНИМ рейтингом (напр. все
  SAVE, чтобы оказаться на одной вкладке Library) через `seed_db.seed_with_comment`
  (тот же механизм, что уже используют TC-014/017/043/045):
  - W1: `tags=["fluff", "hurt-comfort"]` (оба тега)
  - W2: `tags=["fluff"]` (только один из двух)
  - W3: `tags=["canon-divergent"]` (ни одного из выбранных)
- Открыт экран Library, соответствующая вкладка, фильтр не применён — видно все 3
  работы.

## Сценарий (Given-When-Then)

**Given** на вкладке Library видно 3 работы: W1 с тегами fluff+hurt-comfort, W2
только с fluff, W3 без общих тегов

**When** пользователь в фильтр-панели выбирает ОБА тега "fluff" И "hurt-comfort"
(две отметки, не одну) и жмёт "Apply filters"

**Then** список показывает только W1 — единственную работу, у которой присутствуют
ОБА выбранных тега
**And** W2 скрыта, несмотря на то что содержит один из двух выбранных тегов
(частичное пересечение не проходит — фильтр не является OR)
**And** W3 скрыта полностью (ни одного совпадения)

**Инвариант:** личные теги фильтруют по AND: показанное множество работ — всегда
подмножество исходного списка вкладки, и КАЖДАЯ показанная работа содержит ВСЕ
выбранные теги без исключения; работы с частичным (но не полным) пересечением
скрываются наравне с работами без единого совпадения — иначе фильтр вырождался бы
в OR (обобщение банкового пункта «фильтр downloaded-only даёт подмножество Library»
на предикат из N тегов вместо одного булева поля).

## Заметки для автоматизации
- Сидинг — `seed_db.seed_with_comment([(work, rating, None, json.dumps([...]))...])`
  (уже используется `test_backup_restore.py`/TC-021 для tags — см. `_ROWS` в этом
  файле, `json.dumps(["fluff", "found-family"])` — тот же формат). Инфраструктура
  готова, блокера нет.
- Локатор тега в фильтр-панели — `TagChip` в секции "Tags" (`LibraryScreen.kt`,
  `FilterSheetContent`): чипы из `topTags`/`allTags` кликабельны напрямую по тексту
  тега, либо через поле "Search tags" (`tagQuery`) → подсказка в выпадающем списке
  → клик. Если тег не входит в `topTags` (топ-10 по частоте use across всей
  библиотеки — здесь всего 2 разных тега, войдут оба), чип виден без ввода в поиск;
  сверить с живым деревом на случай расхождения.
- Не путать с TC-045 (личные теги НЕ влияют на видимость независимо от фильтрации —
  про рейтинговую видимость, не про сам фильтр тегов) и с TC-028
  (downloaded-only — булев фильтр, здесь предикат из N значений с AND).
