# TC-031 — Граничное значение: work без word_count в сортировке по word count

## Предусловия
- Приложение запущено, в Room засеяны 3 работы с одним рейтингом: две с валидным
  `word_count` (800, 4200), одна с `word_count=null` (сидится через
  `Work(word_count=None)`, см. `framework/data/works.py::NULL_WORD_COUNT_TARGET` —
  блокер снят, AT-BUG-010, Fixed 2026-07-17).
- Открыт экран Library, соответствующая вкладка.

## Сценарий (Given-When-Then)

**Given** на вкладке Library 3 работы: две с word_count (800 и 4200), одна без
word_count (null)

**When** пользователь выбирает сортировку "Word count" (любое направление, high-to-low
или low-to-high)

**Then** работа без word_count отображается последней в списке независимо от
направления сортировки
**And** две остальные работы упорядочены между собой корректно по выбранному
направлению

## Проверяемые данные
| Параметр | Значение |
|---|---|
| Работы | 2 с word_count (800, 4200) + 1 с word_count=null, один рейтинг |

## Заметки для автоматизации
- **Блокер снят (test-maintainer, 2026-07-17, AT-BUG-010 → Fixed):**
  `Work.word_count` теперь типизирован `int | None`; `Work(word_count=None)`
  доходит до колонки `wordCount` (nullable `INTEGER`, см. `AppDatabase.kt`) как
  NULL через обычный sqlite3-bind в `seed_db._insert_rows` — отдельной функции
  не потребовалось (в отличие от `rating=NULL`, которому нужна была
  `_insert_rows_full`/`seed_with_comment`, т.к. `rating` дополнительно
  проверяется по `_RATING_ENUM`). Готовая фикстура —
  `framework/data/works.py::NULL_WORD_COUNT_TARGET`; заводится тем же
  `seed()`/`seed_library`, что обычные работы.
  Device-free доказательство механизма —
  `framework/tests/test_seed_null_wordcount_unit.py`.
- Смежная зависимость `rating=NULL` (TC-014/TC-017) была закрыта РАНЕЕ и
  независимо (`seed_with_comment`, до AT-BUG-010) — проверено при взятии этого
  долга, см. AT-BUG-010 «Обсуждение».
- Кейс готов к автоматизации (test-automator) — блокера в сидинге больше нет;
  собственно ведение по UI (drag сортировки Library) этой правкой не
  проверялось, только сидинг-механизм.
