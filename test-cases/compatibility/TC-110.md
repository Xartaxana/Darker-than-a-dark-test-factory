# TC-110 — Базовый smoke-путь в системном dark и системном light режиме

## Предусловия
- Приложение установлено, данные очищены.
- Работа W засеяна (переиспользовать seed TC-009).
- Системный режим переключается через `adb shell cmd uimode night yes|no`
  (существующий примитив `framework/core/adb.py::set_night_mode`) — это
  СИСТЕМНЫЙ режим ОС, независимый от собственной темы приложения (side
  panel toggle) — пересечение с R-11 в том, что оба влияют на рендер
  WebView/Compose, но переключаются разными путями.

## Сценарий (Given-When-Then)

**Given** приложение запущено с чистыми данными, работа W засеяна

**When** базовый smoke-путь (запуск → Browse страницы W → рейтинг → Library →
таб-операция, тот же маршрут, что TC-109/TC-098) проходится дважды: один раз
с системным `cmd uimode night yes` (dark), один раз с `cmd uimode night no`
(light) — каждый прогон на чистых данных (`pm clear` между прогонами)

**Then** маршрут завершается тем же наблюдаемым результатом в ОБОИХ системных
режимах: рейтинг сохраняется и виден, работа появляется в нужной вкладке
Library, таб открывается/закрывается без краша — независимо от направления
переключения (day→night и night→day)

**Инвариант:** переключение системного dark/light режима не меняет
поведенческий результат базового пути (URL, активная вкладка, сохранённые
рейтинги остаются согласованными с действиями пользователя) — свойство
верно для ОБОИХ членов семьи «темы» (dark, light), не только для одного;
прямое применение стартового банка C4-инвариантов («смена темы не меняет
URL, активную вкладку и сохранённые данные») к системному (не
внутриприложенческому) переключателю темы.

## Проверяемые данные
| Параметр | Значение |
|---|---|
| Системные режимы | `cmd uimode night yes` (dark), `cmd uimode night no` (light) |
| Работа | ao3_id из `data/works.py` (та же, что TC-009) |

## Заметки для автоматизации
- Блокера нет: `adb.set_night_mode(dark: bool)` — уже существующий примитив
  `framework/core/adb.py`, используется рутинно (не требует нового кода);
  smoke-маршрут переиспользует шаги TC-001/006/007/098 — не изобретать новый.
- Пересечение с R-11 (хрупкая тема-зона, см. `app-under-test/CLAUDE.md`
  «Dark mode — fragile area») — при падении сверить, не является ли причина
  уже известным классом дефекта тема-подсистемы, прежде чем заводить как
  новый.
