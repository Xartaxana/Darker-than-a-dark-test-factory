# TC-109 — Базовый smoke-путь на API 26 (нижняя граница minSdk)

## Предусловия
- Второй AVD с нижним API level (нижняя граница — minSdk 26 по манифесту),
  rootable-образ без Google Play (для mitm-CA, см. docs/01 §4) —
  **инфраструктурная предпосылка, на момент дизайна физически ОТСУТСТВУЕТ**
  в `tools/avd` (только `ao3_test_api34.ini`) — см. «Заметки для
  автоматизации» ниже, блокер заведён test_debt-багом.
- Работа W засеяна тем же способом, что на основном AVD (переиспользовать
  seed TC-009/TC-006/TC-007).

## Сценарий (Given-When-Then)

**Given** приложение установлено и запущено на втором AVD (API 26) с чистыми
данными, работа W засеяна

**When** пользователь проходит базовый smoke-путь: запуск → Browse (страница
работы W) → простановка рейтинга → Library (вкладка соответствующего
рейтинга) → открытие/закрытие вкладки (переиспользовать шаги TC-001/006/007,
тот же приём, что TC-098/TC-105 — не изобретать новый маршрут)

**Then** каждый шаг маршрута завершается тем же наблюдаемым результатом, что
на основном AVD (API 34): рейтинг сохраняется и виден без reload, работа
появляется в соответствующей вкладке Library, таб открывается/закрывается без
краша — WebView и Compose UI рендерятся корректно на нижней границе minSdk

## Проверяемые данные
| Параметр | Значение |
|---|---|
| API level (второй AVD) | 26 (minSdk, нижняя граница) |
| API level (основной AVD, база сравнения) | 34 |
| Работа | ao3_id из `data/works.py` (та же, что TC-009) |

## Заметки для автоматизации
- **Блокер автоматизации (инфраструктурный, не устраняется дизайном):**
  второй AVD физически отсутствует в `tools/avd` (проверено: только
  `ao3_test_api34.ini`) — нет rootable-образа API 26 без Google Play,
  необходимого и для запуска приложения на нижней границе minSdk, и для
  установки mitm-CA в replay-режиме (та же предпосылка, что §4 докс/01 для
  записи). Кейс НЕ автоматизируем до появления второго AVD.
- Заведён `bugs/AT-BUG-024.md` (`type: test_debt`, `debt_kind:
  missing_fixture`, `test_cases: [TC-109]`) в этом же ходе — решение
  test-designer: правило CLAUDE.md «блокер в заметках → баг в том же ходе»
  применено буквально, несмотря на указание диспетчера отложить заведение
  бага на test-automator (см. запись в «Обсуждение» самого бага): прецедент
  AT-BUG-004/005/006 (блокеры жили только прозой заметок 2026-07-02→
  2026-07-08/09) — тот самый класс, который правило существует
  предотвращать, воспроизводить намеренно не стал.
- Дизайн кейса НЕ требует наличия AVD (сам сценарий и критерии определены
  сейчас); диспатч автоматизации — только после Fixed по AT-BUG-024.
