# TC-013 — Disliked скрыт при включённой фильтрации

## Предусловия
- Приложение запущено, в Room засеяна работа W с `rating=DISLIKE`
  (`framework/data/seed_db.py`, `DISLIKED` из `framework/data/works.py`).
- В Settings "Enable filtering" включён (дефолт), Disliked входит в hidden-ratings
  set (дефолтное правило из PROJECT.md §Priority rules).
- Открыта листинговая страница (replay-фикстура), которая среди прочих содержит
  блёрб работы W.

## Сценарий (Given-When-Then)

**Given** приложение запущено, работа W засеяна с рейтингом Disliked, фильтрация
включена и Disliked в hidden-set

**When** пользователь открывает листинговую страницу, содержащую блёрб работы W

**Then** блёрб работы W не отображается на странице (скрыт JS-бриджем,
`applyAllFilters`)

**Инвариант:** любая работа, чей `rating` входит в hidden-ratings set при
включённой фильтрации, исключена из рендера листинга — свойство скрытия по
членству в множестве, не привязанное к конкретному `ao3_id` работы W; кейс
проверяет свойство на одном представителе (Disliked), полный охват по
рейтингам дополняют сиблинг-кейсы TC-014 (негатив — что НЕ скрывается) и
TC-015 (фильтрация выключена).

## Проверяемые данные
| Параметр | Значение |
|---|---|
| Работа W | `DISLIKED` из `framework/data/works.py`, засеяна с `rating=DISLIKE` |

## Заметки для автоматизации
- Сидинг через `seed_db.py` перед стартом приложения (данные должны быть в БД до
  первого рендера листинга, `applyAllFilters` читает `window.__ao3HiddenRatings`
  на каждом page load).
- Требует replay-фикстуры листинга, где присутствует блёрб с `ao3_id` работы W
  (совпадение `li#work_{id}` с засеянным `ao3Id`).
- Selector-риск (R-02): скрытие завязано на DOM-структуру `li.work.blurb` — если
  живой AO3 изменит разметку, кейс упадёт по SITE_CHANGED, не APP_BUG — учитывать
  при триаже.

## Заблокировано (test-automator, 2026-07-02)

**Снято (Lead, 2026-07-10):** AT-BUG-004 Verified 2026-07-09 — инфраструктура готова, блокировка неактуальна; ниже — исторический текст. Кейс в очереди Review→Approved (решение человека).

Возвращён в `Review` — невозможно закодировать имеющимися средствами.

**Причина:** кейс требует «replay-фикстуру листинга … содержащую блёрб работы W»
с `li#work_{ao3Id}` для засеянного синтетического `ao3_id`. На реальном
archiveofourown.org таких работ не существует (id намеренно синтетические, см.
`framework/data/works.py`), поэтому единственный способ получить в WebView
листинг с контролируемым блёрбом — replay записанного HTML через mitmproxy.

Replay-транспорт не готов:
- `framework/data/recordings/` — пустая директория, ни одной записи не существует.
- Спайк B (`docs/environment-setup.md`) закрыт только частично: CA/root-механизм
  и расшифровка TLS доказаны, но **захват трафика эмулятора на этом Windows-хосте
  пока не работает** (TCP до mitmdump доходит, обмен не завершается — подозревается
  Windows Defender Firewall).
- фазовая карта (`docs/09` Часть I; на момент записи — `docs/04-roadmap.md`, слито 2026-07-17) Фаза 1 явно фиксирует: «Спайк B — не блокирует каркас
  фреймворка — Фаза 1 строится в live-режиме, replay подключается позже».
- `framework/core/mitm.py` содержит только интерфейс запуска/остановки процесса,
  подключения к прогонам ещё нет (`core/contexts.in_webview` не знает о mitm).

Live-режим не подходит для этого кейса: `BrowserScreen.open_work`/навигация по
живому AO3 не может показать листинг с блёрбом синтетической работы
`900000005`, которой не существует на сайте.

**Что нужно, чтобы разблокировать:**
1. Довести спайк B (правило Windows Firewall для mitmdump/python.exe — нужны
   права администратора, см. `docs/environment-setup.md` §«Что нужно от
   окружения перед Фазой 1») и подключить `core/mitm.py` к `conftest`/`contexts`
   для реального replay-режима.
2. Записать (или вручную сконструировать валидный HTML по образцу AO3 разметки,
   см. `framework/web/selectors.py`) листинговую фикстуру, содержащую
   `li#work_900000005.work.blurb`, и положить в `framework/data/recordings/`.

Область помечена нуждающейся в доводке replay-инфраструктуры — не APP_BUG,
не TEST_BUG; инфраструктурный блокер.
