# TC-107 — Font scaling 1.3× на основных экранах: нет краша, контролы присутствуют

## Предусловия
- Приложение НЕ запущено; система в дефолтном `font_scale`.
- Работа W засеяна с рейтингом (переиспользовать seed TC-009).
- `adb shell settings put system font_scale 1.3` применяется ДО старта
  приложения (простейший детерминированный способ подхватить масштаб — не
  зависит от live-конфига активности).

## Сценарий (Given-When-Then)

**Given** `font_scale` системы установлен в `1.3`, приложение запущено с
чистыми данными и открытой работой W

**When** пользователь проходит представительный smoke-путь по основным
экранам (тот же приём, что TC-098/TC-105 — переиспользовать шаги, не
изобретать новый маршрут): Library → Browse (страница работы W, панель
рейтинга раскрыта) → Settings

**Then** приложение НЕ крашится ни на одном из трёх экранов (процесс жив,
`adb shell pidof <pkg>` возвращает PID на каждом шаге; logcat без `FATAL
EXCEPTION`)
**And** ключевые контролы каждого экрана присутствуют в accessibility tree и
кликабельны: bottom nav (Library/Browse/Settings), Rate-кнопка/панель
рейтинга, слайдеры темы/шрифта/яркости в Settings

## Проверяемые данные
| Параметр | Значение |
|---|---|
| `font_scale` | `1.3` |
| Работа | ao3_id из `data/works.py` (та же, что TC-009) |
| Экраны маршрута | Library, Browse (work page), Settings |

## Заметки для автоматизации
- Блокера нет: `adb.shell("settings put system font_scale 1.3")` — прямое
  расширение существующего примитива `framework/core/adb.py::shell()`, тот же
  класс вызова, что уже использует `set_night_mode`. Восстановление
  `font_scale` в дефолт (`1.0`) после теста — обязательная часть
  teardown/фикстуры (иначе следующий тест унаследует изменённый масштаб,
  класс проблемы — как «logcat не очищен» в TC-105/TC-098).
- **Testability gap (явно названо в docs/01-test-strategy.md §9 area E1):**
  программная детекция ОБРЕЗКИ текста/вёрстки (clipping/overflow) при
  увеличенном шрифте штатным Appium-инструментарием не измеряется. Ассерт
  ограничен присутствием контролов в дереве + отсутствием краша (+
  опциональный скриншот для ручного просмотра на триаже) — НЕ доказывает
  отсутствие визуальной обрезки текста, только отсутствие функционального
  регресса (краш/пропажа контрола из дерева).
- Смок-путь переиспользует шаги TC-001/006/007 (bottombar nav, панель
  рейтинга) — не изобретать новый маршрут.
