# TC-036 — Delete work удаляет файл и запись целиком

## Предусловия
- Приложение запущено, в Room засеяна работа с рейтингом Loved и связанным локальным
  файлом (`downloadPath` заполнен, файл существует).
- Открыт экран Library, вкладка FAVORITE содержит эту работу.

## Сценарий (Given-When-Then)

**Given** работа W имеет рейтинг Loved и скачанный файл

**When** пользователь делает long-press по карточке работы W и в overlay выбирает
"Delete work"

**Then** файл удаляется с диска
**And** работа W полностью исчезает из Library — не отображается ни на вкладке
FAVORITE, ни на вкладке FILES

## Проверяемые данные
| Параметр | Значение |
|---|---|
| Работа W | засеяна с rating=SAVE и заполненным downloadPath на существующий файл |

## Заметки для автоматизации
- Пара к TC-035 (Delete downloaded file) — вместе они покрывают оба destructive
  действия overlay; намеренно не объединены в один кейс (разные Then).
- Проверка "исчезла из Library" — по обеим вкладкам, т.к. это единственный способ
  косвенно убедиться, что строка `WorkRating` удалена целиком (нет прямого доступа к
  БД из UI-теста).
