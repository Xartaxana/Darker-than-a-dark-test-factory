# TC-011 — Мгновенное обновление бейджа (вход: листинг)

## Предусловия
- Приложение запущено с чистыми данными.
- Открыта листинговая страница (replay-фикстура) с работой W, ещё без рейтинга.

## Сценарий (Given-When-Then)

**Given** приложение запущено и открыта листинговая страница с работой W без рейтинга
**And** зафиксирован baseline: WebView не совершал повторную навигацию с момента
загрузки листинга

**When** пользователь через Rate-кнопку работы W и bottom-sheet ставит рейтинг «Liked»

**Then** стиль/цвет Rate-кнопки работы W на карточке блёрба меняется на отражающий
«Liked» немедленно, в пределах той же отрисованной страницы
**And** WebView не выполнил повторную навигацию/reload (тот же history entry, нет
нового `onPageFinished`)

## Проверяемые данные
| Параметр | Значение |
|---|---|
| Работа W | синтетическая работа из replay-фикстуры листинга |

## Заметки для автоматизации
- Аналогично TC-010, но точка входа — листинг/bottom-sheet, а не встроенная панель;
  оставлены раздельными кейсами, т.к. §9 стратегии явно требует "проверить оба входа".
- Технический сигнал "без reload" согласовать с test-automator (см. заметку TC-010).

- [x] Given описывает полное состояние, воспроизводимое фикстурами
- [x] Then проверяет наблюдаемое поведение, а не реализацию
- [x] Указаны приоритет, область и источник требования
