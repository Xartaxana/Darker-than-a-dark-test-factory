# TC-053 — Pinch/spread на странице меняет размер шрифта

## Предусловия
- Приложение запущено с чистыми данными, вкладка Browse активна, открыта страница
  AO3 с читаемым текстом, размер шрифта на среднем (не граничном) шаге.
- Side panel свёрнут (жест выполняется по области контента).
- Режим прогона: replay либо live-smoke.

## Сценарий (Given-When-Then)

**Given** на Browse открыта страница AO3 с текстом среднего размера

**When** пользователь выполняет двухпальцевый spread (разведение пальцев) поверх
страницы, превышая порог распознавания жеста

**Then** размер текста веб-страницы увеличивается (тот же эффект, что кнопка «A+» в
side panel), появляется временный HUD-индикатор уровня шрифта, и обратный жест pinch
(сведение пальцев) уменьшает размер обратно

## Проверяемые данные
| Параметр | Значение |
|---|---|
| Жест увеличения | двухпальцевый spread |
| Жест уменьшения | двухпальцевый pinch |
| Побочный признак | HUD-индикатор уровня шрифта появляется на время жеста |

## Заметки для автоматизации
- Жест классифицируется как «шрифт» при изменении расстояния между пальцами
  (span) ≥ 30dp; шаг ≈ 50dp span на одну ступень. Параллельный вертикальный
  драг двумя пальцами (без изменения span) — это яркость (см. TC-055), не шрифт:
  чтобы попасть в font-ветку, менять именно span (разводить/сводить пальцы), а не
  двигать оба пальца синхронно вниз/вверх.
- Технически сложный жест для Appium/UiAutomator2: нужен multi-touch pinch с
  контролируемым span (`W3C Actions`, два pointer'а). Держать смещение по span
  заметно больше порога 30dp, но не срываться в вертикальный параллельный драг
  (порог яркости 20dp по totalDy срабатывает раньше — если пальцы поедут вниз
  синхронно, жест уйдёт в яркость). Развод по горизонтали безопаснее вертикали.
- Тот же `fontSizeStep`, что кнопки панели и Settings (третий вход в один стейт);
  границы 0..6 clamp'ятся так же (см. TC-052). Наблюдать относительное изменение
  размера текста, не абсолютные px.
- **Реализовано:** встроенные UiAutomator2-жесты `mobile: pinchOpenGesture`/
  `pinchCloseGesture` разводят/сводят два пальца по диагонали от центра области —
  это меняет span (dy0/dy1 разных знаков → avgDy=0 → totalDy не растёт), поэтому
  надёжно попадает в font-ветку, а не в brightness; проверено на живом дереве
  (SharedPreferences `font_size_step` меняется в нужную сторону, без побочного
  срабатывания яркости). Раздельная реализация multi-touch через сырые W3C Actions
  не потребовалась.
- **Известное ограничение:** HUD `LevelIndicator` (BrowserScreen.kt ~370–380)
  рендерится без `contentDescription`/testTag и виден ~800ms — недостаточно
  наблюдаемый признак для UiAutomator2 без гонки по таймингу. Автотест проверяет
  функциональный контракт (рост/уменьшение размера текста в обе стороны жеста), не
  сам факт появления HUD.
