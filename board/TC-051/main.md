# TC-051 — Font size из side panel применяется мгновенно и персистится

## Предусловия
- Приложение запущено с чистыми данными, `fontSizeStep` в дефолтном значении,
  вкладка Browse активна, открыта страница AO3 с читаемым текстом.
- Side panel развёрнут.
- Режим прогона: replay (детерминированная страница AO3) либо live-smoke.

## Сценарий (Given-When-Then)

**Given** на Browse открыта страница AO3 с текстом заданного размера, side panel
развёрнут с кнопками «A-» и «A+»

**When** пользователь нажимает «A+» (увеличение шрифта) один раз, затем полностью
перезапускает приложение (kill + relaunch) и снова открывает ту же страницу

**Then** текст веб-страницы становится крупнее сразу после нажатия «A+» (без
перезагрузки страницы), и после перезапуска приложения размер шрифта остаётся
увеличенным (значение сохранено в настройках)

## Проверяемые данные
| Параметр | Значение |
|---|---|
| Контрол | «A+» в side panel |
| Ожидание сразу | текст крупнее без reload страницы |
| Ожидание после рестарта | увеличенный размер сохранён |

## Заметки для автоматизации
- Наблюдаемость размера шрифта: сравнить относительный размер текстового блока
  (высота/ширина элемента или скриншот-сравнение) до и после — абсолютные px не
  фиксировать. textZoom применяется мгновенно (`LaunchedEffect(fontSizeStep)`,
  без reload) — проверять без ожидания загрузки страницы.
- Персистентность — через `SharedPreferences` `font_size_step`; воспроизводимый
  рестарт: `adb shell am force-stop` + relaunch (не `pm clear` — очистит настройку).
- Тот же `fontSizeStep`, что в Settings и в жесте pinch/spread — здесь вход именно
  через кнопки панели. Эквивалентность входов — TC-054; жест — TC-052.
