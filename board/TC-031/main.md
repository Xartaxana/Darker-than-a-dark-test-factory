# TC-031 — Граничное значение: work без word_count в сортировке по word count

## Предусловия
- Приложение запущено, в Room засеяны 3 работы с одним рейтингом: две с валидным
  `word_count` (800, 4200), одна с `word_count=null` (не заполнено при сидинге —
  требует доработки `seed_db.py`, см. заметки).
- Открыт экран Library, соответствующая вкладка.

## Сценарий (Given-When-Then)

**Given** на вкладке Library 3 работы: две с word_count (800 и 4200), одна без
word_count (null)

**When** пользователь выбирает сортировку "Word count" (любое направление, high-to-low
или low-to-high)

**Then** работа без word_count отображается последней в списке независимо от
направления сортировки
**And** две остальные работы упорядочены между собой корректно по выбранному
направлению

## Проверяемые данные
| Параметр | Значение |
|---|---|
| Работы | 2 с word_count (800, 4200) + 1 с word_count=null, один рейтинг |

## Заметки для автоматизации
- `seed_db.py::_insert_rows` требует явного `word_count` через `Work` dataclass
  (дефолт 1000, не null) — для null-значения нужно расширение сидинга (передать
  `None` явно и убедиться, что INSERT принимает NULL для wordCount). Похожая
  зависимость уже отмечена для `rating=NULL` в TC-014/TC-017 — расширение сидинга,
  видимо, потребуется в нескольких P1-кейсах сразу; стоит сделать один раз.
- Без доработки сидинга этот кейс не автоматизируем как есть — до автоматизации
  оставить `status: Review` (дизайн кейса корректен и независим от готовности
  инструмента), при необходимости пометить блокером в docs/02 (архитектура
  фреймворка) отдельно от этого файла.
