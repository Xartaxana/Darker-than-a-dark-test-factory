# TC-049 — Тема System переключается вместе с системной темой ОС

## Предусловия
- Приложение запущено с чистыми данными, в Settings выбрана тема "System".
- Системная тема Android установлена в Light (`adb shell cmd uimode night no` или
  эквивалент).

## Сценарий (Given-When-Then)

**Given** в приложении выбрана тема "System", системная тема ОС — Light, нативные
экраны приложения отображаются светлыми

**When** системная тема ОС переключается на Dark (`adb shell cmd uimode night yes`)

**Then** нативные Compose-экраны приложения отображаются в тёмной схеме без
дополнительных действий пользователя внутри приложения (следование за System,
`configChanges="uiMode"` не пересоздаёт Activity — состояние сохраняется)

## Проверяемые данные
| Параметр | Значение |
|---|---|
| Тема приложения | System (не менялась в Settings) |
| Системная тема | Light → Dark через adb |

## Заметки для автоматизации
- `adb shell cmd uimode night yes/no` — стандартный способ переключения системной
  тёмной темы на эмуляторе/устройстве без захода в системные настройки UI.
- Отличается от TC-047/TC-048 (явное переключение темы внутри приложения) — здесь
  тема приложения не меняется, меняется внешний фактор (ОС), на который приложение
  обязано реагировать через режим System.
- WebView-часть (см. TC-048, требует холодного старта) здесь не проверяется отдельно
  — фокус на нативном Compose UI и самом факте следования System; если потребуется
  комбинированная проверка System+WebView+cold start, это отдельный P2-кейс (не
  запрошен явно в §9).
