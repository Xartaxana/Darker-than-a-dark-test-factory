# TC-027 — Фильтр word count min/max сужает список работ

## Предусловия
- Приложение запущено, в Room засеяны работы с разным `word_count`, все с одним
  рейтингом (напр. все PENDING, чтобы оказаться на одной вкладке): 300, 800, 1500,
  4200, 12000 слов (`framework/data/works.py::ALL`, засеяны с одинаковым рейтингом
  через `seed_db.py`).
- Открыт экран Library, вкладка с этими работами (напр. PENDING), фильтр не применён.

## Сценарий (Given-When-Then)

**Given** на вкладке Library видно 5 работ с word_count 300/800/1500/4200/12000

**When** пользователь открывает фильтр-панель и задаёт диапазон word count
min=1000, max=5000

**Then** список показывает только работы с word_count в диапазоне [1000, 5000]
(1500 и 4200) — 2 работы
**And** работы с 300, 800 и 12000 не отображаются

**Инвариант:** после применения диапазонного фильтра множество видимых работ
равно {work | min ≤ word_count ≤ max} — точное подмножество исходных 5
работ по предикату диапазона (фильтр не удаляет работы внутри диапазона и
не добавляет работы вне него).

## Проверяемые данные
| Параметр | Значение |
|---|---|
| Работы | `ALL` из `framework/data/works.py`, засеяны с одинаковым рейтингом |
| word_count values | 300, 800, 1500, 4200, 12000 |
| Фильтр | min=1000, max=5000 |

## Заметки для автоматизации
- **2026-07-17 (C4-ретрофит, test-designer) — пробел в покрытии границ
  инварианта:** ни одно из засеянных значений word_count
  (300/800/1500/4200/12000) не равно точно `min=1000` или `max=5000`,
  поэтому включительность границ диапазона (`≤`, а не `<`) не проверяется ни
  этим Then, ни `test_library_filter_word_count_range`
  (`framework/tests/test_library_filters.py:16`). Свойство «точное
  подмножество по предикату» доказано для значений строго внутри/вне
  диапазона, но не на самих границах. Не блокер автоматизации (не
  отсутствующая фикстура/сидинг — значения уже есть в наборе, нужна лишь
  доработка диапазона фильтра или доп. work с word_count=1000/5000) —
  design-note в очередь test-automator, `test_debt`-багом не заводится.
- **2026-07-18 (C4-ретрофит закрыт, test-maintainer) — дыра закрыта.**
  Сверено по коду (`LibraryScreen.kt:164-169`, инлайн-фильтр в composable —
  подпись файла исправлена на приёмке по вердикту critic): фильтр сравнивает
  `wordCount >= min` и `wordCount <= max` — ОБЕ границы включительные, не
  строгие. Добавлены два work'а с word_count РОВНО 1000 и РОВНО 5000
  (`framework/data/works.py::WORD_COUNT_MIN_BOUNDARY`/`MAX_BOUNDARY`),
  засеваемые вместе с `ALL` отдельной фикстурой
  `library_word_count_boundary_seeded` (`framework/tests/conftest.py`) —
  НЕ добавлены в `library_all_one_rating_seeded`, чтобы не задеть состав
  данных TC-029. `test_library_filter_word_count_range` теперь assert'ит
  видимость обоих граничных work'ов в диапазоне [1000, 5000], закрывая
  включительность на самой границе, не только строго внутри/вне.
- Все 5 работ должны иметь один и тот же рейтинг при сидинге (не 5 разных, как в
  дефолтном `ALL`), иначе они окажутся на разных вкладках и фильтр нечего будет
  сравнивать на одной вкладке — потребуется параметр rating при вызове `seed()`.
  Уточнить сигнатуру `seed_db.seed(works: list[tuple[Work, str]])` — передать один и
  тот же rating-строку для всех пяти.
- Фильтр применяется клиентски, сортировка — после фильтра; здесь сортировку не
  менять (оставить дефолт Last read), чтобы не смешивать с TC-030/TC-031.
- Локатор фильтр-панели/полей min/max — из `LibraryFilters`/`LibraryScreen.kt`, место
  рендера bottom sheet или inline-панели; сверить с живым деревом.
