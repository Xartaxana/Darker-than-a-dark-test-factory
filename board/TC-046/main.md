# TC-046 — Кастомная error page с Retry при ошибке загрузки

## Предусловия
- Приложение запущено с чистыми данными.
- Активная вкладка настроена на переход по заведомо недоступному URL (напр.
  несуществующий хост в replay-режиме, где mitmproxy отвечает ошибкой соединения,
  либо офлайн-режим сети на эмуляторе для конкретного домена).

## Сценарий (Given-When-Then)

**Given** активная вкладка пытается загрузить URL, недоступный по сети (main-frame
запрос завершается ошибкой)

**When** WebView получает `onReceivedError` для главного фрейма

**Then** отображается кастомная themed error page приложения (не дефолтная страница
ошибки Chrome/WebView) с текстом об ошибке
**And** на странице присутствует ссылка/кнопка Retry, ведущая на исходный (упавший)
URL
**And** нажатие Retry инициирует повторную загрузку того же URL (наблюдаемо —
повторная попытка сети, а не переход на другой адрес)

## Проверяемые данные
| Параметр | Значение |
|---|---|
| URL | заведомо недоступный (напр. `https://nonexistent.invalid.test/`) |

## Заметки для автоматизации
- Воспроизведение ошибки сети предпочтительно через контролируемый способ (mitmproxy
  replay без записи для домена → connection refused, либо `adb shell svc wifi
  disable` кратковременно для конкретного сценария) — не полагаться на реальную
  нестабильность живого AO3 (R-03), это сделает тест флаки.
- Error page загружена через `about:blank` как base URL — следствие: она не попадает
  в историю навигации. Отдельная проверяемая деталь: после Retry (успешного или нет)
  кнопка "назад" не должна показывать саму error page как отдельный шаг истории —
  это можно вынести в отдельный P2-кейс, если понадобится; здесь не проверяется,
  чтобы не смешивать с основным сценарием (один сценарий — один кейс).
- Также не ловит bridge-инъекцию на самой error-странице (PROJECT.md) — если
  тестовая инфраструктура пытается искать bridge-элементы (Rate-кнопки и т.п.) на
  error page, это неверный локатор по определению; не проверять их отсутствие как
  отдельный assert здесь (не относится к пользовательскому сценарию, это
  implementation detail), разве что понадобится для устойчивости самого теста.
