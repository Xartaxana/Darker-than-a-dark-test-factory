# TC-103 — file:// доступ: попытка выйти за пределы контента загрузки

## Предусловия
- В Room засеяна работа со связанным локальным `.html` файлом (тот же механизм
  `seed_with_download`, что TC-034), но HTML-фикстура ДОПОЛНЕНА (сверх базовой
  фикстуры TC-034) одной тестовой ссылкой `<a id="probe-link"
  href="file:///data/user/0/com.example.ao3_wrapper/databases/ao3.db">probe</a>`
  — целевой путь реально существует на устройстве (тот же UID, внутренняя БД
  приложения), но заведомо НЕ является частью пользовательского «скачанного
  контента».
- Открыт экран Library, вкладка FILES содержит эту работу.

## Сценарий (Given-When-Then)

**Given** на вкладке FILES есть работа со связанным `.html`-файлом, содержащим
встроенную ссылку на `file://`-путь ВНЕ директории загрузок (внутренний файл
приложения, не являющийся частью скачанного контента)

**When** пользователь открывает работу (тап по карточке, как в TC-034), затем
внутри открывшейся страницы тапает по тестовой ссылке `probe-link`

**Then** WebView НЕ отображает содержимое целевого файла как успешно
загруженную страницу — либо навигация остаётся на прежнем URL, либо
показывается ошибка загрузки/пустой результат; открытие скачанного контента не
превращается в произвольный файловый браузер по внутреннему хранилищу
приложения по клику на встроенную в HTML ссылку

## Проверяемые данные
| Параметр | Значение |
|---|---|
| Работа | фикстурный `.html`, как в TC-034, + добавленная тестовая `file://`-ссылка на путь вне директории загрузок |
| Целевой путь ссылки | реально существующий internal-файл приложения (напр. Room DB), заведомо вне `_DOWNLOAD_FIXTURE_REL_DIR` |
| Ожидаемый результат | навигация по ссылке не приводит к успешно отображённому содержимому целевого файла |

## Заметки для автоматизации
- Блокера нет: `seed_with_download`/`_push_download_fixture`
  (`framework/data/seed_db.py`) уже поддерживают произвольный локальный HTML —
  фикстура TC-034 расширяется одной дополнительной ссылкой, инфраструктура не
  меняется. `browser_steps`/`library_steps` открытия файла — переиспользовать
  из TC-034, тап по ссылке — обычное Appium-взаимодействие с WebView-элементом.
- **Пересечение с TC-034 (∩), не дублирование:** TC-034 — ПОЗИТИВНЫЙ кейс
  (штатная стилизация штатного скачанного файла); этот кейс — НЕГАТИВНЫЙ/граничный
  (попытка уйти за пределы контента загрузки через встроенную ссылку). Общая
  инфраструктура открытия файла переиспользуется, assert'ы разные, не
  дублируются.
- **Testability gap (частично drivable, как явно названо в docs/01-test-strategy.md
  §9 area E4).** Полная модель прав SAF (OS-уровневые URI-permissions,
  предоставленные пользователем через Storage Access Framework) недоступна
  для инспекции через UiAutomator2 black-box — эта проба целится в конкретный
  наблюдаемый вектор (WebView file://-навигация по клику внутри уже открытого
  контента), не в исчерпывающую модель прав. Целевой путь ссылки в этом кейсе —
  файл ВНУТРИ песочницы того же приложения (тот же UID, ОС не блокирует чтение
  сама по себе) — кейс проверяет поведение ПРИЛОЖЕНИЯ (не превращает открытие
  скачанного контента в файловый браузер), не гарантию ОС. Проверка обхода
  SAF-разрешений на файлы ДРУГИХ приложений/системных путей вне скоупа
  минимального смоука (потребовала бы иной, более глубокой атаки, ближе к
  полному аудиту, §8).
