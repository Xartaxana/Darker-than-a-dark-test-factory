# TC-038 — Auto-scan (silent) при смене папки загрузок перелинковывает файл

## Предусловия
- Приложение запущено; в новой (целевой) SAF-папке загрузок заранее лежит `.html`
  файл с именем, соответствующим паттерну `buildFilename` (оканчивается на
  `_<workId>.html`, где workId совпадает с `ao3_id` засеянной работы).
- В Room засеяна работа с этим `ao3_id`, рейтингом, но `downloadPath=null` (файл
  считается потерянным/не привязанным — напр. после переустановки).
- Открыт экран Settings, текущая папка загрузок отличается от целевой.

## Сценарий (Given-When-Then)

**Given** в целевой папке лежит orphan-файл `..._<workId>.html`, соответствующая
работа в Room существует, но без `downloadPath`

**When** пользователь меняет папку загрузок в Settings на целевую через SAF-пикер

**Then** запускается silent-скан автоматически (без явного нажатия "Scan for
downloads")
**And** появляется диалог результата, так как что-то было перелинковано (найден и
привязан 1 файл)
**And** работа в Library получает связанный файл — карточка показывает open-иконку

## Проверяемые данные
| Параметр | Значение |
|---|---|
| Работа | засеяна с известным `ao3_id`, `downloadPath=null` |
| Файл | `<любой_префикс>_<ao3_id>.html` в целевой SAF-папке |

положен adb-шеллом во вложенную SAF-подпапку; папка загрузок сменена через
`saf_steps.saf_pick_folder`; проверены ОБА наблюдаемых Then-пункта кейса —
диалог «Scan complete» с точным текстом
(`"Found 1 files — relinked 1, added 0 new."`, подтверждает, что перелинковано
именно ПОТОМУ ЧТО что-то реально найдено, не просто "диалог появился") И
open-иконка карточки в Library (`library_steps.assert_open_icon_shown`) — оба,
не один на выбор, раз оба явно названы в Then кейса. 3 зелёных прогона подряд
(включая один прогон, упавший на сидинге известным классом AT-BUG-009
`adb shell am start -W` timeout — перезапущен без правок `seed_db.py`, не
относится к SAF-области).

Побочная находка (задокументирована в `bugs/AT-BUG-005.md`, эта же дата, НЕ
блокирует этот кейс): `DocumentsUIScreen.reset_to_root()` может таймаутить,
если OpenDocumentTree повторно открывается на ТОЙ ЖЕ подпапке, что была
подтверждена непосредственно перед этим тем же тестовым процессом — обойдено
на уровне теста уникальным именем подпапки на каждый вызов фикстуры
(`uuid4()`-суффикс), инфраструктура SAF не менялась.
