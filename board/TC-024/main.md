# TC-024 — Эвикция самого старого снапшота при 6-м закрытии подряд

## Предусловия
- Приложение запущено с чистыми данными.
- Открыто 7 вкладок с различимыми URL (например разные страницы работ), достаточно
  для 6 последовательных закрытий и минимум одной оставшейся активной вкладки.

## Сценарий (Given-When-Then)

**Given** открыто 7 вкладок с различимыми URL

**When** пользователь последовательно закрывает 6 вкладок по одной (swipe-up по чипу
каждый раз, snackbar каждый раз просто исчезает/сменяется следующим без нажатия Undo)

**Then** после 6-го закрытия snackbar с Undo относится к 6-й (самой последней)
закрытой вкладке
**And** нажатие Undo восстанавливает именно 6-ю закрытую вкладку
**And** самая первая (1-я) из шести закрытых вкладок больше не восстановима — её
снапшот вытеснен из истории (проверяется невозможностью откатиться к ней дальше
одного доступного Undo-снапшота на момент проверки)

## Проверяемые данные
| Параметр | Значение |
|---|---|
| Вкладки до закрытий | 7, с различимыми URL/заголовками |
| Число последовательных закрытий | 6 |
| Лимит истории закрытий | 5 (`up to 5 most-recent closes are undoable`) |

## Заметки для автоматизации
- Прямой способ проверить "невосстановимость" 1-й закрытой вкладки — ограничен UI:
  snackbar показывает Undo только для последнего закрытия. Assert строится на том,
  что после Undo текущего (6-го) снапшота новый snackbar для следующего отката не
  появляется автоматически (Undo не "цепочка"), либо (если реализовано иначе) на
  количестве вкладок после серии undo-действий не превышающем ожидаемое. Уточнить
  фактическое поведение по живому UI перед реализацией теста — возможна корректировка
  формулировки Then без потери сути риска (эвикция старых снапшотов).
- Общий риск R-08 ("персистентность табов, undo") — этот кейс покрывает граничное
  значение лимита истории, отдельно от TC-023 (happy path одного undo).
