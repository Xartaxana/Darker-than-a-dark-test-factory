# TC-061 — Free-text search по Library ищет по title/author/fandom/tags/comment

## Предусловия
- Приложение запущено, в Room засеяны 3 работы с ОДНИМ рейтингом через
  `seed_db.seed_with_comment`:
  - W1: `comment="Reread this every wintersong"` (уникальная подстрока только здесь)
  - W2: обычные title/author/fandom, `comment=null`, `tags=null` — не содержит
    подстроку ни в одном поле
  - W3: аналогично W2, другой title/author/fandom — тоже не содержит подстроку
- Открыт экран Library, соответствующая вкладка, фильтр не применён — видно все 3
  работы.

## Сценарий (Given-When-Then)

**Given** на вкладке Library видно 3 работы, только у W1 в поле `comment` есть
подстрока "wintersong"

**When** пользователь вводит "WinterSong" (смешанный регистр) в поле "Search any
field" фильтр-панели и жмёт "Apply filters"

**Then** список показывает только W1 (совпадение найдено в `comment`, поиск
регистронезависим)
**And** W2 и W3 скрыты (подстрока не встречается ни в одном из полей title/author/
fandom/tags/comment)
**And** очистка поля поиска (крестик) и повторное применение возвращают список ко
всем 3 работам

**Инвариант:** свободный текстовый поиск сужает список Library до подмножества, где
для КАЖДОЙ оставшейся работы query встречается хотя бы в ОДНОМ из полей (title/
author/fandom/tags/comment) без учёта регистра; работы без совпадения ни в одном
поле полностью скрыты — тот же класс, что фильтр downloaded-only (банк: «фильтр
даёт подмножество Library»), применённый к предикату OR по нескольким текстовым
полям вместо одного булева.

## Проверяемые данные
| Параметр | Значение |
|---|---|
| Работы | 3, один рейтинг, различающиеся title/author/fandom/comment |
| W1.comment | `"Reread this every wintersong"` |
| Query | `"WinterSong"` (проверка регистронезависимости) |

## Заметки для автоматизации
- Сидинг — `seed_db.seed_with_comment`, как в TC-060/TC-021 — инфраструктура готова,
  блокера нет.
- Локатор поля поиска — `InputField` с `placeholder = "Search any field"` в
  `FilterSheetContent` (иконка `Icons.Default.Search`), trailing-иконка очистки
  видна только при непустом значении — сверить с живым деревом.
- Отличается от TC-060 семантикой: здесь OR по нескольким полям одного запроса, там
  AND по нескольким выбранным тегам — не смешивать assert'ы в одном тесте.
- Не проверяет visual-подсветку совпадения на карточке (`highlightText` в
  `WorkCard`) — это деталь реализации подсветки, не предмет этого кейса о видимости
  списка; если понадобится отдельный P2/P3-кейс на подсветку, не в этом файле.
