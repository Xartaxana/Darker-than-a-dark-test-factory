# TC-077 — Tag-кнопка присутствует ⟺ есть личные теги вне AO3-набора карточки (replay)

## Предусловия
- Приложение запущено, режим **replay** (`listing_basic.mitm`).
- Работа `DISLIKED` (900000005) засеяна с `rating=DISLIKE` и личным тегом,
  заведомо отсутствующим в записанной разметке карточки `DISLIKED`
  (`seed_db.py::seed_with_comment`, поле `tags`); остальные 4 работы без
  личных тегов.

## Сценарий (Given-When-Then)

**Given** приложение запущено в replay-режиме, `DISLIKED` засеяна с личным
тегом вне её AO3-тегов, открыта `listing_basic.mitm`

**When** страница полностью загружается (`applyRatings` применяет засеянные
данные)

**Then** `[data-ao3-btn-wrap]` работы `DISLIKED` содержит `[data-ao3-tag-btn]`
**And** остальные 4 работы (без личных тегов) НЕ имеют `[data-ao3-tag-btn]`

**Инвариант:** тот же контракт, что TC-076, детерминированно проверенный на
смешанном наборе (1 работа с личным тегом вне её AO3-набора + 4 без личных
тегов на одной странице).

## Проверяемые данные
| Параметр | Значение |
|---|---|
| Фикстура | `framework/data/recordings/listing_basic.mitm` |
| Работа | `DISLIKED` (900000005), личный тег вне AO3-набора карточки |

## Заметки для автоматизации
- `seed_with_comment` уже поддерживает `tags` независимо от `rating`/`comment`
  (AT-BUG-004, инкремент 1) — блокеров нет.
- Тот же дефект гранулярности реестра, что в TC-076 — не повторяю доклад
  здесь, см. заметку TC-076.
- Маркер: `@pytest.mark.p0 @pytest.mark.replay`.
