# TC-076 — Tag-кнопка присутствует ⟺ есть личные теги вне AO3-набора карточки (live)

## Предусловия
- Приложение запущено с чистыми данными, режим **live**.
- Открыта реальная листинговая страница AO3 с ≥1 работой.

## Сценарий (Given-When-Then)

**Given** приложение запущено live, открыта реальная листинговая страница AO3
с работой W (не оценена, без личных тегов)

**When** пользователь нажимает Rate-кнопку работы W, в `RatingOverlay`
добавляет личный тег, заведомо отсутствующий среди AO3-тегов карточки W
(`ul.tags.commas li a.tag`), сохраняет

**Then** та же `[data-ao3-btn-wrap]` работы W получает `[data-ao3-tag-btn]`
**And** если добавленный личный тег СОВПАДАЕТ (без учёта регистра) с одним из
AO3-тегов, УЖЕ показанных на карточке W, `[data-ao3-tag-btn]` НЕ появляется
(`getCustomTags` фильтрует пересечение с `ao3Set`)

**Инвариант:** присутствие Tag-кнопки — биусловное свойство относительно
РАЗНОСТИ множеств (личные теги работы \ AO3-теги её же карточки), не факта
«есть ли вообще личные теги» — свойство верно для любого состава личных
тегов, а не только для примера с одним заведомо новым тегом.

## Проверяемые данные
| Параметр | Значение |
|---|---|
| Работа W | первая видимая работа реального листинга |
| Личный тег | текст, заведомо отсутствующий в `ul.tags.commas` карточки W |

## Заметки для автоматизации
- Дробление реестра (замечено при проектировании, докладывается в отчёте
  test-strategist'у): `bridge-rate-note-tag-buttons` в
  `docs/feature-registry.yaml` бандлит Rate/Note/Tag тремя разными Then
  (unrated-присутствие / comment-биусловность / custom-tag-биусловность) под
  одним заголовком с "/" — тот же класс, что урок hide/dim. Этот кейс
  (вместе с TC-070/071/074/075/072/073) закрывает все три режима явно;
  предлагаю test-strategist раздробить запись на 3 id при следующей
  инвентаризации, не блокирую этим дизайн.
- Не требует известного `ao3_id`.
- Маркер: `@pytest.mark.p0 @pytest.mark.live`.
- Сиблинг-кейс TC-077 — тот же контракт в replay.
