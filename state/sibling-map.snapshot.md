Источник правды — `D:\Improving_AI\Operating-System-for-LLMs\docs\SIBLING_MAP.md`
(живая карта, репо Operating-System-for-LLMs). Этот файл — закоммиченный
СРЕЗ для сред, где живая карта структурно недостижима (облачная сессия:
Linux-контейнер без `D:\`): `scripts/mechanism_gate.py` берёт его
ТРЕТЬИМ звеном цепочки разрешения (env `AO3_SIBLING_MAP` → живая карта →
этот срез) и печатает факт использования в stderr коммита.

Несущее в этом файле — ТОЛЬКО номера осей (`## Ось N`): гейт проверяет
полноту осевого блока перечислением номеров. Названия — best-effort
конспект по осевым блокам механизменных коммитов этого репо (срез
2026-07-23, соответствует блоку коммита 22e7b855, 9 осей); за точной
семантикой оси — живая карта.

Дисциплина среза (правило 10в — детектор): сверка среза с живой картой
— чек еженедельной калибровки OS-репо (заведён HANDOFF'ом в очередь
калибровки №4); Windows-сессия, заметившая новую ось в живой карте,
обновляет срез ТЕМ ЖЕ ходом (файл под самозащитой гейта — правка
требует осевой блок + tier). Расхождение среза с живой картой,
пропустившее механизменный коммит, = находка калибровки (класс F-48:
носитель, который целевой деплой не читает, испаряется).
